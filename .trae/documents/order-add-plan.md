# 订单列表新增订单功能 - 开发计划

## 项目背景
根据用户需求，需要完善 `j:\shadcn-admin\src\features\orders\OrderList.tsx` 的新增订单功能，参考 `j:\FastAPI-Shadcn-Admin\frontend\src\pages\OrderManagement.tsx` 的实现，并按照 `j:\shadcn-admin\src\features\users` 的目录结构组织代码。

## 功能需求
1. 点击新增订单按钮，弹出新增订单窗口
2. 窗口上半部包含：
   - 客户名称 ComboBox（下拉选择客户，可搜索）
   - 订单编号
   - 订单日期（自动生成当前时间）
   - 交货日期（日期选择框）
3. 下半部是可供编辑的订单子项目表格，每一行包括：
   - 规格
   - 产品类型
   - 型号
   - 数量
   - 单位
   - 销售单价
   - 备注
   - 合同编号
   - 客户物流编号
   - 外购（单选框，默认不选择）
4. 默认有一行可供编辑
5. 弹出窗口有一个新增产品按钮，按一下增加一行
6. 窗口最下方有取消和保存按钮
7. 保存过程：
   - 先将订单编号、客户名称、订单日期、交货日期保存至数据库的订单列表
   - 然后再保存订单子项目（包含订单编号、客户名称、订单日期、交货日期）至数据库的订单表
   - 同时将订单表的 oid 字段设置为订单列表中该订单编号的 id

## 技术实现细节

### 1. 订单新增对话框组件
- 使用 `Dialog` 组件实现对话框
- 使用 `Input` 组件实现表单输入
- 使用 `DatePicker` 组件实现日期选择
- 使用 `Table` 组件实现订单子项目表格
- 支持添加和删除订单子项目

### 2. OrderList 组件修改
- 添加 `showAddModal` 状态
- 添加 `addFormData` 状态，存储新增订单的表单数据
- 添加 `addOrderItems` 状态，存储新增订单的子项目数据
- 添加 `handleAddOrder` 函数，打开新增订单对话框
- 添加 `handleSaveNewOrder` 函数，保存新增订单

### 3. API 调用
- 使用 `orderAPI.createOrder` 创建订单
- 使用 `orderAPI.createOrderItem` 创建订单子项目
- 先创建订单，获取订单 ID，然后创建订单子项目

### 4. 代码组织
- 按照 users 目录的结构，将组件放在 `components` 目录下
- 将数据相关的代码放在 `data` 目录下
