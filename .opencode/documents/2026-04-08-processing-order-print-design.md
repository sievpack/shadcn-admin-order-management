# 加工单PDF打印功能 - 设计文档

## 1. 需求概述

### 1.1 目标
将现有的加工单打印功能从HTML模板迁移到PDF模板，提升打印效果和专业度。

### 1.2 模板文件
- **PDF模板**：`D:\shadcn-admin\src\template\加工单模板.pdf`

### 1.3 数据来源
订单分项数据（订单模块的 Order 表）

## 2. 技术方案

### 2.1 前端渲染方案
使用 `react-pdf` 在前端渲染PDF模板，叠加动态数据后调用浏览器打印。

**技术栈**：
- `react-pdf`：PDF渲染库
- 现有前端架构：React + TanStack Query + shadcn/ui

**优点**：
- 原生PDF效果，跨浏览器兼容性好
- 数据与模板分离，便于维护
- 前端处理，减轻后端压力

## 3. 数据字段映射

### 3.1 订单分项 → PDF字段

| PDF字段 | 数据来源 | 说明 |
|---------|----------|------|
| 规格 | 订单分项.规格 | 直接映射 |
| 产品类型 | 订单分项.产品类型 | 直接映射 |
| 宽度 | 型号第一减号之前 | 如 "10-20-30" → "10" |
| 长度 | 型号第二减号之后 | 如 "10-20-30" → "30" |
| 节距 | 字典数据 sync_belt_pitch | 根据产品类型查字典获取 |
| 数量 | 订单分项.数量 | 直接映射 |
| 单位 | 订单分项.单位 | 直接映射 |
| 备注 | 订单分项.备注 | 直接映射 |

### 3.2 特殊字段处理

**型号解析规则**：
```
型号格式："宽度-长度-节距" 或 "宽度-长度"
示例："10-20-30" → 宽度=10, 长度=30, 节距=30
      "10-20"    → 宽度=10, 长度=20, 齿数=查字典
```

**节距获取**：

- 根据"产品类型"查字典 `sync_belt_pitch`
- 字典的 `dict_label` 与产品类型匹配

## 4. PDF布局设计

### 4.1 单页布局（每页5行）

```
┌─────────────────────────────────────────────┐
│  工单编号: [自动生成]          第X页/共X页   │
├─────────────────────────────────────────────┤
│ # │ 产品类型 │ 规格 │ 宽度 │ 长度 │ 齿数 │ 数量 │ 单位 │ 备注 │
├───┼─────────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ 1 │         │      │      │      │      │      │      │      │
├───┼─────────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┤
│   │ 特殊情况说明（合并行）                                      │
├───┼─────────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ 2 │         │      │      │      │      │      │      │      │
├───┼─────────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┤
│   │ 特殊情况说明（合并行）                                      │
... (共5行)                                        │
└───────────────────────────────────────────────────────────────┘
```

### 4.2 分页规则
- 每页最多显示5条订单分项
- 每条分项占2行（数据行 + 特殊情况合并行）
- 超出的分项自动分到下一页
- 页脚显示"第X页/共X页"

**获取加工单打印数据**

```
GET /api/order/processing-print/{order_id}
```

**请求参数**：
- `order_id`: 订单ID

**响应数据**：

## 7. 实现步骤

### Phase 1: 基础设施
1. 安装 `react-pdf` 依赖
2. 配置 PDF worker
3. 创建PDF渲染基础组件

### Phase 2: 数据层
1. 后端新增 `/api/order/processing-print/{order_id}` 接口
2. 前端创建 `useProcessingOrderPrint` Query Hook
3. 实现型号解析工具函数

### Phase 3: 组件开发
1. 创建 `ProcessingOrderPDF.tsx` 组件
2. 实现PDF模板加载和渲染
3. 实现动态文本叠加

### Phase 4: 集成
1. 在订单分项页面添加"打印加工单"按钮
2. 集成打印预览功能
3. 添加 loading 状态
