## 重构模式参考

参考 `orders` 模块的重构模式：

```
旧模式:
useEffect + fetchData → setState → 手动刷新

新模式:
TanStack Query Hooks → useQuery/useMutation → invalidateQueries + refreshKey
```

### 必须遵循的原则

1. **数据层分离**：Query hooks 管理服务端数据，本地 state 管理 UI 状态
2. **mutationFn 必须返回数据**：返回 `response.data` 而非 `undefined`
3. **父子组件同步**：使用 `refreshKey` 机制
4. **组件拆分**：Dialog 按功能拆分（add/edit/delete），表格组件抽取展开行

## 技术备忘

### mutationFn 必须返回数据

```tsx
// ✅ 正确
mutationFn: async (data) => {
  const response = await api.create(data)
  return response.data
}

// ❌ 错误
mutationFn: (data) => api.create(data)
```

### refreshKey 机制

```tsx
// 父组件
const [refreshKey, setRefreshKey] = useState(0)
const handleSave = async () => {
  await createMutation.mutateAsync(data)
  setRefreshKey(k => k + 1)
}

// 子组件
useEffect(() => {
  if (refreshKey > 0) {
    refetch()
  }
}, [refreshKey])
```
