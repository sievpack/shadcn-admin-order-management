# 搜索框输入抖动问题修复

## 问题描述

发货管理模块的工具栏搜索框在输入时页面出现严重抖动和闪烁，尤其是中文输入时体验极差。

## 问题原因

### 根本原因

1. **频繁状态更新**：`toolbar.tsx` 中 `handleSearch` 每次 `onChange` 都立即调用 `onSearch`，导致：
   - 每次输入都触发 URL 更新
   - URL 更新触发 React Query 重新查询
   - 页面频繁重新渲染

2. **双重渲染**：
   - `setLocalSearchValue` 更新本地状态 → Input 重渲染
   - `onSearch` 触发 URL 更新 → 整个页面状态变化

### 问题代码

```tsx
// 修改前的 handleSearch
const handleSearch = useCallback(
  (value: string) => {
    if (serverPaginationMode) {
      setLocalSearchValue(value)  // ❌ 每次输入都更新状态
      if (onSearch) {
        onSearch(value)           // ❌ 每次输入都触发查询
      }
      return
    }
    // ...
  },
  [serverPaginationMode, onSearch]
)
```

## 解决方案

### 方案思路

1. **区分中文输入状态**：使用 `isComposingRef` 跟踪 `onCompositionStart` 和 `onCompositionEnd` 事件
2. **优化搜索触发时机**：
   - 中文输入：只在 `onCompositionEnd`（输入完成）时触发一次
   - 英文输入：每次 `onChange` 都触发搜索（因为没有输入法组合问题）
   - 清空输入框：直接触发搜索重置页面

### 核心代码

```tsx
// 1. 添加输入状态跟踪 ref
const isComposingRef = useRef(false)

// 2. Input 事件处理
onChange={(event) => {
  if (serverPaginationMode) {
    const value = event.target.value
    setLocalSearchValue(value)
    if (isComposingRef.current) {
      return  // 中文输入过程中不触发搜索
    }
    onSearch?.(value)  // 非组合输入时，每次都触发
  } else {
    handleSearch(event.target.value)
  }
}}

onCompositionStart={() => {
  isComposingRef.current = true
}}

onCompositionEnd={(event) => {
  isComposingRef.current = false
  if (serverPaginationMode) {
    const value = event.target.value
    setLocalSearchValue(value)
    onSearch?.(value)  // 中文输入完成时触发
  }
}}

onKeyDown={
  serverPaginationMode
    ? (event) => {
        if (event.key === 'Enter') {
          handleSearch((event.target as HTMLInputElement).value)
        }
      }
    : undefined
}
```

## 经验总结

### 1. 中文输入处理要点

| 事件 | 含义 | 处理方式 |
|------|------|----------|
| `onCompositionStart` | 中文输入法开始组合 | 设置 `isComposingRef = true` |
| `onChange` (组合中) | 组合文本变化 | 不触发业务逻辑，只更新本地状态 |
| `onCompositionEnd` | 中文输入法完成 | 触发业务逻辑，清空组合状态 |

### 2. React Input 性能优化

- **受控 vs 非受控**：频繁更新的搜索框应避免完全受控导致的频繁重渲染
- **避免在 onChange 中做重操作**：将业务逻辑与状态更新分离
- **中文输入特殊性**：需要区分组合输入状态，避免拼音字母被当作搜索关键词

### 3. 搜索框设计模式

对于服务器端搜索框，推荐的交互模式：

```
英文输入 → 每次 onChange → 触发搜索
中文输入 → 开始(不触发) → 输入中(不触发) → 结束(onCompositionEnd) → 触发搜索
清空输入 → onChange(value='') → 触发搜索重置页面
```
