# 前后端接口不一致问题修复记录

## 问题描述
前端调用 UPDATE 接口（如订单分项编辑）提示成功，但实际数据未更新。

## 根本原因
后端 API 层缺少 `db.commit()`，导致数据未真正持久化到数据库。

## 修复范围

### 1. 后端 API 修复（添加 db.commit() + db.refresh()）

| 模块 | 文件 | 修复内容 |
|------|------|----------|
| 订单分项 | `backend/app/api/order/item.py` | UPDATE 接口添加 commit + refresh |
| 客户 | `backend/app/api/customer.py` | CREATE/UPDATE 接口添加 commit + refresh |
| 生产计划 | `backend/app/api/production/plan.py` | UPDATE 返回完整数据 |
| 生产工单 | `backend/app/api/production/order.py` | UPDATE 返回完整数据 |
| 质检记录 | `backend/app/api/production/qc.py` | UPDATE 返回完整数据 |
| 成品入库 | `backend/app/api/production/inbound.py` | UPDATE 返回完整数据 |
| 报价单 | `backend/app/api/quote.py` | UPDATE 返回完整数据 |
| 应收账款 | `backend/app/api/finance/__init__.py` | UPDATE 返回完整数据 |
| 用户 | `backend/app/api/user.py` | UPDATE 返回完整数据 |

### 2. 前端 TanStack Query 修复

| 文件 | 修复内容 |
|------|----------|
| `src/queries/orders/useUpdateOrderItem.ts` | mutationFn 返回 response.data |
| `src/queries/orders/useUpdateOrder.ts` | mutationFn 返回 response.data |

### 3. 前端刷新机制修复

| 文件 | 修复内容 |
|------|----------|
| `src/features/customers/CustomerList.tsx` | 实现 handleRefresh 刷新逻辑 |
| `src/features/customers/components/customer-table.tsx` | 连接 setRefreshData 回调 |

## 修复模式

```
API 层更新操作 → Service 调用 Repository → Repository 更新 + flush
                                              ↓
                                    API 层必须调用 commit
                                              ↓
                                    API 层调用 refresh 获取最新数据
                                              ↓
                                    返回更新后的完整对象
```

## 测试验证

通过 API 测试验证更新确实持久化到数据库：
- 更新前查询 → 修改 → 更新后查询 → 数据已变化

## 经验总结

1. **所有 UPDATE/CREATE 接口必须显式调用 db.commit()**
2. **UPDATE 接口应返回更新后的完整数据，便于前端验证**
3. **TanStack Query mutationFn 应返回 response.data 而非 axios response**
4. **前端刷新机制需要正确连接 onRefresh 和 refreshData 回调**
