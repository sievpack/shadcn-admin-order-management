# 对账周期设计

针对对账周期（如“26-25”表示每月26号到次月25号，“1-30”表示每月1号到30号，但2月需要特殊处理），数据库存储的核心是**将周期规则结构化**，而不是存储字符串。使用以下设计：

## 表结构

```sql
CREATE TABLE dbo.ReconciliationCycle (
    CycleId          INT IDENTITY(1,1) PRIMARY KEY,
    CycleName        NVARCHAR(50)      NOT NULL,          -- '月度(26-25)'
    StartDay         TINYINT           NOT NULL,          -- 起始日 1-31
    EndDay           TINYINT           NOT NULL,          -- 结束日 1-31，用 0 表示月底
    IsEndInclusive   BIT               NOT NULL DEFAULT 1 -- 结束日是否包含在周期内
);
```

## 存储示例

| CycleName          | StartDay | EndDay | IsEndInclusive |
| :----------------- | :------- | :----- | :------------- |
| 26号→次月25号      | 26       | 25     | 1              |
| 1号→30号（含30）   | 1        | 30     | 1              |
| 1号→月底（含月底） | 1        | 0      | 1              |
| 16号→15号（跨月）  | 16       | 15     | 1              |

`EndDay = 0` 表示该月的最后一天（利用 `EOMONTH`）

## 核心 T-SQL 函数：根据给定日期计算所属对账周期

### 1. 获取周期的起始日期

```sql
CREATE OR ALTER FUNCTION dbo.GetCycleStartDate (
    @RefDate DATE,
    @StartDay TINYINT
)
RETURNS DATE
AS
BEGIN
    DECLARE @StartDate DATE = DATEFROMPARTS(YEAR(@RefDate), MONTH(@RefDate), @StartDay);
    -- 如果起始日无效（如4月31日），取当月最后一天
    IF @StartDate IS NULL
        SET @StartDate = EOMONTH(@RefDate);
    -- 如果起始日大于参考日，说明周期起始在上个月
    IF @StartDate > @RefDate
        SET @StartDate = DATEADD(MONTH, -1, @StartDate);
    RETURN @StartDate;
END;
```

### 2. 获取周期的结束日期

```sql
CREATE OR ALTER FUNCTION dbo.GetCycleEndDate (
    @StartDate DATE,
    @EndDay TINYINT,
    @IsEndInclusive BIT
)
RETURNS DATE
AS
BEGIN
    DECLARE @EndDate DATE;
    IF @EndDay = 0  -- 月底
        SET @EndDate = EOMONTH(@StartDate);
    ELSE
        SET @EndDate = DATEFROMPARTS(YEAR(@StartDate), MONTH(@StartDate), @EndDay);
    
    -- 处理跨月：如果结束日 < 起始日（且不是月底特殊值），结束日在下个月
    IF @EndDay != 0 AND @EndDay < DAY(@StartDate)
        SET @EndDate = DATEADD(MONTH, 1, @EndDate);
    
    -- 处理无效日期（如2月30号 → 2月最后一天）
    IF @EndDate IS NULL
        SET @EndDate = EOMONTH(DATEADD(MONTH, 1, @StartDate)); -- 保守处理
    
    -- 如果不包含结束日，则返回前一天
    IF @IsEndInclusive = 0
        SET @EndDate = DATEADD(DAY, -1, @EndDate);
    
    RETURN @EndDate;
END;
```

### 3. 根据任意日期查出它属于哪个周期（起始日～结束日）

```sql
CREATE OR ALTER FUNCTION dbo.GetCycleRange (
    @CheckDate DATE,
    @StartDay TINYINT,
    @EndDay TINYINT,
    @IsEndInclusive BIT
)
RETURNS @Range TABLE (StartDate DATE, EndDate DATE)
AS
BEGIN
    DECLARE @StartDate DATE, @EndDate DATE;
    -- 先假设起始月为 @CheckDate 所在月
    SET @StartDate = dbo.GetCycleStartDate(@CheckDate, @StartDay);
    SET @EndDate   = dbo.GetCycleEndDate(@StartDate, @EndDay, @IsEndInclusive);
    
    -- 如果 @CheckDate 不在 [StartDate, EndDate] 内，尝试前一个周期
    IF @CheckDate < @StartDate OR @CheckDate > @EndDate
    BEGIN
        SET @StartDate = DATEADD(MONTH, -1, @StartDate);
        SET @EndDate   = dbo.GetCycleEndDate(@StartDate, @EndDay, @IsEndInclusive);
    END;
    
    INSERT INTO @Range VALUES (@StartDate, @EndDate);
    RETURN;
END;
```

### 使用示例

```sql
-- 查询 '2025-02-20' 属于哪个周期（StartDay=26, EndDay=25）
SELECT * FROM dbo.GetCycleRange('2025-02-20', 26, 25, 1);
-- 返回 StartDate = '2025-01-26', EndDate = '2025-02-25'

-- 查询 '2025-02-28' 在周期 1-30 中（30号不存在，自动取月底）
SELECT * FROM dbo.GetCycleRange('2025-02-28', 1, 30, 1);
-- StartDate = '2025-02-01', EndDate = '2025-02-28'（因为2月只有28天）
```

## 查询优化建议：预计算周期边界表（物化视图）

如果对账记录表很大，每次查询都调用函数会较慢。创建一个**周期边界预计算表**：

```sql
CREATE TABLE dbo.CycleBoundaries (
    CycleId       INT,
    CycleStartDate DATE,
    CycleEndDate   DATE,
    PRIMARY KEY (CycleId, CycleStartDate)
);

-- 每日或每月运行存储过程，为未来N年预生成边界
CREATE OR ALTER PROCEDURE dbo.GenerateCycleBoundaries
    @YearsForward INT = 5
AS
BEGIN
    DECLARE @StartDate DATE = DATEFROMPARTS(YEAR(GETDATE()), 1, 1);
    DECLARE @EndDate   DATE = DATEADD(YEAR, @YearsForward, @StartDate);
    DECLARE @Current   DATE = @StartDate;
    
    WHILE @Current <= @EndDate
    BEGIN
        INSERT INTO dbo.CycleBoundaries (CycleId, CycleStartDate, CycleEndDate)
        SELECT 
            c.CycleId,
            dbo.GetCycleStartDate(@Current, c.StartDay),
            dbo.GetCycleEndDate(@Current, c.EndDay, c.IsEndInclusive)
        FROM dbo.ReconciliationCycle c;
        
        SET @Current = DATEADD(DAY, 1, @Current);
    END;
END;
```

之后查询对账记录时直接 JOIN 这个预计算表，性能最佳。

## 边界情况处理（SQL Server 特有）

| 情况                                   | SQL Server 处理方式                            |
| :------------------------------------- | :--------------------------------------------- |
| 起始日 31 且当月无 31                  | `DATEFROMPARTS` 返回 NULL，使用 `EOMONTH` 替代 |
| 结束日 30 且 2 月                      | 同样用 `EOMONTH` 返回 2 月最后一天             |
| 跨年周期（如 2025-12-26 → 2026-01-25） | `DATEADD(MONTH, 1, ...)` 自动处理年份递增      |
| 包含/不包含结束日                      | 用 `DATEADD(DAY, -1, EndDate)` 调整            |

## 结合结算周期表多表联查

### 1. 假设的表结构

```sql
-- 客户表（含对账周期ID）
CREATE TABLE Customer (
    CustomerId INT PRIMARY KEY,
    Name NVARCHAR(100),
    CycleId INT NOT NULL  -- 引用 ReconciliationCycle
);

-- 对账周期规则表（之前设计的）
CREATE TABLE ReconciliationCycle (
    CycleId INT PRIMARY KEY,
    CycleName NVARCHAR(50),
    StartDay TINYINT NOT NULL,   -- 起始日 1-31
    EndDay TINYINT NOT NULL,     -- 结束日 1-31, 0表示月底
    IsEndInclusive BIT NOT NULL DEFAULT 1
);

-- 订单表（含发货日期和金额）
CREATE TABLE Orders (
    OrderId INT PRIMARY KEY,
    CustomerId INT NOT NULL,
    OrderDate DATE NOT NULL,
    ShipDate DATE,               -- 发货日期，NULL表示未发货
    Amount DECIMAL(18,2) NOT NULL,
    FOREIGN KEY (CustomerId) REFERENCES Customer(CustomerId)
);

-- 发货表（可选，如果订单表已有ShipDate则不需要；如果一次订单多次发货，则需要单独发货表）
-- 为简化，假设订单表ShipDate即全部发货日期，且一个订单只发一次货。

-- 收款表（用于计算已收款金额）
CREATE TABLE Payments (
    PaymentId INT PRIMARY KEY,
    CustomerId INT NOT NULL,
    PaymentDate DATE NOT NULL,
    Amount DECIMAL(18,2) NOT NULL,
    -- 可选关联订单：OrderId INT
    FOREIGN KEY (CustomerId) REFERENCES Customer(CustomerId)
);
```

### 2. 核心函数：获取周期起止日期（SQL Server）

如果未创建，先执行以下函数（兼容上面设计的周期规则）：

```sql
-- 获取周期的起始日期
CREATE OR ALTER FUNCTION dbo.GetCycleStartDate (
    @RefDate DATE,
    @StartDay TINYINT
)
RETURNS DATE
AS
BEGIN
    DECLARE @StartDate DATE = DATEFROMPARTS(YEAR(@RefDate), MONTH(@RefDate), @StartDay);
    IF @StartDate IS NULL
        SET @StartDate = EOMONTH(@RefDate);  -- 无效日（如4月31）取当月最后一天
    IF @StartDate > @RefDate
        SET @StartDate = DATEADD(MONTH, -1, @StartDate);
    RETURN @StartDate;
END;
GO

-- 获取周期的结束日期
CREATE OR ALTER FUNCTION dbo.GetCycleEndDate (
    @StartDate DATE,
    @EndDay TINYINT,
    @IsEndInclusive BIT
)
RETURNS DATE
AS
BEGIN
    DECLARE @EndDate DATE;
    IF @EndDay = 0
        SET @EndDate = EOMONTH(@StartDate);
    ELSE
        SET @EndDate = DATEFROMPARTS(YEAR(@StartDate), MONTH(@StartDate), @EndDay);
    
    -- 跨月处理：结束日 < 起始日（且非月底）则移到下月
    IF @EndDay != 0 AND @EndDay < DAY(@StartDate)
        SET @EndDate = DATEADD(MONTH, 1, @EndDate);
    
    IF @EndDate IS NULL
        SET @EndDate = EOMONTH(DATEADD(MONTH, 1, @StartDate));
    
    IF @IsEndInclusive = 0
        SET @EndDate = DATEADD(DAY, -1, @EndDate);
    
    RETURN @EndDate;
END;
GO
```

### 3. 预计算周期边界表（性能关键）

为每个客户的每个周期提前生成边界日期范围，避免每次查询都调用函数。

```sql
-- 周期边界表（存储每个客户每个周期的起止日期）
CREATE TABLE dbo.CustomerCycleBoundary (
    CustomerId INT NOT NULL,
    CycleStartDate DATE NOT NULL,
    CycleEndDate DATE NOT NULL,
    PRIMARY KEY (CustomerId, CycleStartDate)
);

-- 生成未来N年的边界（建议每天/每月增量更新）
CREATE OR ALTER PROCEDURE dbo.GenerateCustomerCycleBoundaries
    @YearsForward INT = 5
AS
BEGIN
    -- 清空旧数据（或增量更新）
    TRUNCATE TABLE dbo.CustomerCycleBoundary;
    
    DECLARE @StartDate DATE = DATEFROMPARTS(YEAR(GETDATE()), 1, 1);
    DECLARE @EndDate   DATE = DATEADD(YEAR, @YearsForward, @StartDate);
    DECLARE @CurrDate  DATE = @StartDate;
    
    -- 获取所有客户及其周期规则
    DECLARE @CustomerCycles TABLE (
        CustomerId INT,
        StartDay TINYINT,
        EndDay TINYINT,
        IsEndInclusive BIT
    );
    INSERT INTO @CustomerCycles
    SELECT c.CustomerId, rc.StartDay, rc.EndDay, rc.IsEndInclusive
    FROM Customer c
    JOIN ReconciliationCycle rc ON c.CycleId = rc.CycleId;
    
    -- 为每一天生成所有客户的周期起始（简化：逐日扫描，数据量大时可用月份循环优化）
    WHILE @CurrDate <= @EndDate
    BEGIN
        INSERT INTO dbo.CustomerCycleBoundary (CustomerId, CycleStartDate, CycleEndDate)
        SELECT 
            cc.CustomerId,
            dbo.GetCycleStartDate(@CurrDate, cc.StartDay) AS CycleStartDate,
            dbo.GetCycleEndDate(
                dbo.GetCycleStartDate(@CurrDate, cc.StartDay),
                cc.EndDay,
                cc.IsEndInclusive
            ) AS CycleEndDate
        FROM @CustomerCycles cc;
        
        SET @CurrDate = DATEADD(DAY, 1, @CurrDate);
    END;
    
    -- 去重（同一个客户同一个周期可能重复插入，因为每天都会生成相同的周期）
    WITH Dedup AS (
        SELECT DISTINCT CustomerId, CycleStartDate, CycleEndDate
        FROM dbo.CustomerCycleBoundary
    )
    DELETE FROM dbo.CustomerCycleBoundary;
    INSERT INTO dbo.CustomerCycleBoundary
    SELECT * FROM Dedup;
    
    -- 添加索引
    CREATE INDEX IX_CustomerCycleBoundary_Date ON dbo.CustomerCycleBoundary(CycleStartDate, CycleEndDate) INCLUDE (CustomerId);
END;
GO

-- 执行生成（比如每年执行一次）
EXEC dbo.GenerateCustomerCycleBoundaries @YearsForward = 5;
```

### 4. 查询每月（对账周期）的应收账款

应收账款 = 周期内发货总额 - 周期内收款总额（假设收款抵扣该周期内的应收账款，具体需根据业务规则，这里按收款日期所在周期扣减）

```sql
-- 1. 按周期汇总发货金额
WITH ShipByCycle AS (
    SELECT 
        o.CustomerId,
        cb.CycleStartDate,
        cb.CycleEndDate,
        SUM(o.Amount) AS TotalShippedAmount
    FROM Orders o
    JOIN dbo.CustomerCycleBoundary cb 
        ON o.CustomerId = cb.CustomerId
        AND o.ShipDate BETWEEN cb.CycleStartDate AND cb.CycleEndDate
    WHERE o.ShipDate IS NOT NULL
    GROUP BY o.CustomerId, cb.CycleStartDate, cb.CycleEndDate
),
-- 2. 按周期汇总收款金额
PaymentByCycle AS (
    SELECT 
        p.CustomerId,
        cb.CycleStartDate,
        cb.CycleEndDate,
        SUM(p.Amount) AS TotalPaidAmount
    FROM Payments p
    JOIN dbo.CustomerCycleBoundary cb 
        ON p.CustomerId = cb.CustomerId
        AND p.PaymentDate BETWEEN cb.CycleStartDate AND cb.CycleEndDate
    GROUP BY p.CustomerId, cb.CycleStartDate, cb.CycleEndDate
)
-- 3. 计算应收账款（发货 - 收款）
SELECT 
    s.CustomerId,
    c.Name AS CustomerName,
    s.CycleStartDate,
    s.CycleEndDate,
    s.TotalShippedAmount,
    ISNULL(p.TotalPaidAmount, 0) AS TotalPaidAmount,
    s.TotalShippedAmount - ISNULL(p.TotalPaidAmount, 0) AS Receivables
FROM ShipByCycle s
LEFT JOIN PaymentByCycle p 
    ON s.CustomerId = p.CustomerId 
    AND s.CycleStartDate = p.CycleStartDate
JOIN Customer c ON s.CustomerId = c.CustomerId
ORDER BY s.CustomerId, s.CycleStartDate;
```

### 5. 如果需要对账周期显示为友好名称（如“2025-01-26 至 2025-02-25”）

在查询中添加格式化列：

```sql
CONCAT(FORMAT(CycleStartDate, 'yyyy-MM-dd'), ' 至 ', FORMAT(CycleEndDate, 'yyyy-MM-dd')) AS CycleRange
```

### 6. 性能优化建议

- **索引**：

  ```sql
  CREATE INDEX IX_Orders_ShipDate_Customer ON Orders(ShipDate, CustomerId) INCLUDE (Amount);
  CREATE INDEX IX_Payments_PaymentDate_Customer ON Payments(PaymentDate, CustomerId) INCLUDE (Amount);
  CREATE INDEX IX_Customer_CycleId ON Customer(CycleId);
  ```

- **周期边界表更新策略**：每天凌晨增量生成未来1年的边界（只生成新的日期范围），而不是全量重建。
- **分区视图**：如果数据量极大，可按周期起始日期对 `CustomerCycleBoundary` 进行分区。

## 4. API 端点实现

python

```python
from fastapi import FastAPI, HTTPException, Query
from typing import List
import pyodbc

app = FastAPI()

@app.get("/api/receivables", response_model=List[ReceivableItem])
def get_receivables(
    customer_id: Optional[int] = Query(None, description="客户ID，不传则返回所有"),
    start_cycle: Optional[date] = Query(None, description="周期起始日 >= 此值"),
    end_cycle: Optional[date] = Query(None, description="周期结束日 <= 此值")
):
    """
    按对账周期查询应收账款（发货 - 收款）
    """
    conn = get_db_connection()
    cursor = conn.cursor()
    
    # 基础 SQL（使用预计算表 CustomerCycleBoundary）
    sql = """
    WITH ShipByCycle AS (
        SELECT 
            o.CustomerId,
            cb.CycleStartDate,
            cb.CycleEndDate,
            SUM(o.Amount) AS TotalShippedAmount
        FROM Orders o
        JOIN CustomerCycleBoundary cb 
            ON o.CustomerId = cb.CustomerId
            AND o.ShipDate BETWEEN cb.CycleStartDate AND cb.CycleEndDate
        WHERE o.ShipDate IS NOT NULL
        GROUP BY o.CustomerId, cb.CycleStartDate, cb.CycleEndDate
    ),
    PaymentByCycle AS (
        SELECT 
            p.CustomerId,
            cb.CycleStartDate,
            cb.CycleEndDate,
            SUM(p.Amount) AS TotalPaidAmount
        FROM Payments p
        JOIN CustomerCycleBoundary cb 
            ON p.CustomerId = cb.CustomerId
            AND p.PaymentDate BETWEEN cb.CycleStartDate AND cb.CycleEndDate
        GROUP BY p.CustomerId, cb.CycleStartDate, cb.CycleEndDate
    )
    SELECT 
        s.CustomerId,
        c.Name AS CustomerName,
        s.CycleStartDate,
        s.CycleEndDate,
        s.TotalShippedAmount,
        ISNULL(p.TotalPaidAmount, 0) AS TotalPaidAmount,
        s.TotalShippedAmount - ISNULL(p.TotalPaidAmount, 0) AS Receivables
    FROM ShipByCycle s
    LEFT JOIN PaymentByCycle p 
        ON s.CustomerId = p.CustomerId 
        AND s.CycleStartDate = p.CycleStartDate
    JOIN Customer c ON s.CustomerId = c.CustomerId
    WHERE 1=1
    """
    
    params = []
    
    # 动态添加过滤条件
    if customer_id:
        sql += " AND s.CustomerId = ?"
        params.append(customer_id)
    if start_cycle:
        sql += " AND s.CycleStartDate >= ?"
        params.append(start_cycle)
    if end_cycle:
        sql += " AND s.CycleEndDate <= ?"
        params.append(end_cycle)
    
    sql += " ORDER BY s.CustomerId, s.CycleStartDate"
    
    try:
        cursor.execute(sql, params)
        rows = cursor.fetchall()
        result = [
            ReceivableItem(
                customer_id=row[0],
                customer_name=row[1],
                cycle_start_date=row[2],
                cycle_end_date=row[3],
                total_shipped=float(row[4]),
                total_paid=float(row[5]),
                receivables=float(row[6])
            )
            for row in rows
        ]
        return result
    except pyodbc.Error as e:
        raise HTTPException(status_code=500, detail=f"Database error: {str(e)}")
    finally:
        cursor.close()
        conn.close()
```

## 6. 补充：生成周期边界表的调用

在 FastAPI 启动时，可添加一个后台任务或定时调用存储过程，确保 `CustomerCycleBoundary` 表有最新数据：

@app.on_event("startup")
def init_cycle_boundaries():
    """启动时检查并生成未来2年的周期边界"""
