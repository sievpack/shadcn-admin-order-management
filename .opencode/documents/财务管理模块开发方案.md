## 二、数据库规划

### 表清单 (8张表 + 3个视图)

| # | 表名 | 说明 |
|---|------|------|
| 1 | 应收账款表 | 客户应收款管理 |
| 2 | 收款记录表 | 客户回款记录 |
| 3 | 应收账款核销表 | 应收款核销记录 |
| 4 | 应付账款表 | 供应商应付款管理 |
| 5 | 付款记录表 | 供应商付款记录 |
| 6 | 付款核销表 | 应付款核销记录 |
| 7 | 凭证表 | 会计凭证管理 |
| - | 收入统计视图 | 收入查询 |
| - | 应收账款账龄视图 | 账龄分析 |
| - | 应付账款账龄视图 | 账龄分析 |

### 核心字段设计

**应收账款表**:
- 应收单号 (AR-YYYYMMDD-001)、关联订单、客户名称、应收金额、已收金额、应收余额
- 应收日期、到期日期、账期类型 (预付款/月结30天/货到付款)
- 收款状态 (未收款/部分收款/已结清/坏账)

**收款记录表**:
- 收款单号 (CR-YYYYMMDD-001)、关联应收、收款金额、收款方式 (现金/银行转账/承兑/微信/支付宝)
- 核销状态、操作人

**凭证表**:
- 凭证编号 (VCH-YYYYMMDD-001)、凭证日期、凭证类型
- 摘要、科目、借方金额、贷方金额
- 审核状态 (待审核/已审核/已过账)

## 三、后台接口设计

API 路由: `/api/finance`

| 模块 | 接口 |
|------|------|
| 应收账款 | GET /ar/list, POST /ar/create, PUT /ar/write-off/{id}, GET /ar/aging |
| 收款记录 | GET /collection/list, POST /collection/create, POST /collection/verify |
| 应付账款 | GET /ap/list, POST /ap/create, PUT /ap/write-off/{id}, GET /ap/aging |
| 付款记录 | GET /payment/list, POST /payment/create, POST /payment/verify |
| 凭证管理 | GET /voucher/list, POST /voucher/create, PUT /voucher/approve/{id} |
| 财务报表 | GET /stats/income, GET /stats/profit, GET /stats/balance, GET /stats/cash-flow |
